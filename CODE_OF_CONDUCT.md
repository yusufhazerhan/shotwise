# Code of Conduct

## Our commitment

We want Shotwise to be a welcoming project for people with different backgrounds, experience levels, and communication styles.

That means we aim to make collaboration feel safe, respectful, and useful.

## Expected behavior

- Be respectful and patient.
- Give feedback about the work, not the worth of the person doing it.
- Assume good intent, but stay honest when something needs correction.
- Leave enough context in issues and reviews for someone else to succeed.
- Prefer curiosity over dunking.

## Unacceptable behavior

- Harassment, intimidation, or discrimination
- Personal attacks or insults
- Deliberately hostile review behavior
- Publishing private information without consent
- Trolling, baiting, or repeated bad-faith disruption

## Maintainer responsibilities

Project maintainers may remove, edit, or reject comments, commits, code, issues, and pull requests that are not aligned with this Code of Conduct.

Maintainers may also temporarily or permanently restrict participation for repeated or severe violations.

## Reporting

If you experience or witness unacceptable behavior, please contact the maintainers privately instead of escalating in public.

Until a dedicated project inbox exists, use the security contact path in `SECURITY.md` for sensitive reports and clearly mark the report as a conduct issue.

## Scope

This Code of Conduct applies in:

- GitHub issues
- pull requests
- code review
- project discussions
- project-linked chat spaces and demos
